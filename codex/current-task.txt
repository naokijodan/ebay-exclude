## 公開するツール（5つ）

### 1. list_policies
- 説明: eBayの全フルフィルメントポリシー一覧を取得
- 入力: marketplaceId? (string, default "EBAY_US")
- 処理: getFulfillmentPolicies() を呼んで、各ポリシーのname, id, 除外国数を返す
- 出力: JSON配列 [{name, id, excludedCount}]

### 2. get_policy_exclusions
- 説明: 特定ポリシーの除外設定を人間が読める形で取得
- 入力: policyId (string, 必須)
- 処理: getFulfillmentPolicy() で取得、classifyRegionName() + reverseMapping で表示名に変換
- 出力: JSON {policyName, policyId, exclusions: {regions: [], domestics: [], countries: []}}

### 3. update_exclusions
- 説明: 特定ポリシーの除外設定を更新
- 入力:
  - policyId (string, 必須)
  - addExclusions? (string[], 追加する除外項目の表示名。例: ["Africa", "Japan", "PO Box"])
  - removeExclusions? (string[], 解除する除外項目の表示名)
  - setExclusions? (string[], 除外リストを丸ごと置き換え。表示名の配列)
- 処理:
  1. 現在のポリシーを取得
  2. add/remove/set に応じて除外リストを更新
  3. 表示名→APIName変換（regionMapping, countryToIso を使用）
  4. updateFulfillmentPolicy() で更新
- 出力: JSON {success, policyName, added: [], removed: [], totalExclusions: number}

### 4. export_all_excel
- 説明: 全ポリシーの除外設定をピボット形式のExcelに出力
- 入力: outputPath? (string, default "ebay-policies.xlsx")
- 処理: exportAllCommand() をそのまま呼ぶ（src/commands/export-all.tsからインポート）
- 出力: JSON {success, filePath, policyCount, columnCount}

### 5. import_excel
- 説明: ピボット形式のExcelからeBayに除外設定を反映
- 入力:
  - filePath (string, 必須)
  - dryRun? (boolean, default true — 安全のためデフォルトでドライラン)
- 処理: importAllCommand() をそのまま呼ぶ（src/commands/import-all.tsからインポート）
- 出力: JSON {success, updatedCount, skippedCount, dryRun}

## 実装の注意点

1. dotenv/config を最初にインポート（.envから環境変数を読む）
2. 既存の関数をインポートして使う:
   - `src/lib/ebay-api.ts` → getFulfillmentPolicies, getFulfillmentPolicy, updateFulfillmentPolicy
   - `src/lib/classify-region.ts` → classifyRegionName
   - `src/data/definitions.ts` → regionMapping, countryToIso
   - `src/commands/export-all.ts` → exportAllCommand
   - `src/commands/import-all.ts` → importAllCommand
3. エラーハンドリング: try-catchで包み、isError: trueでエラー返却
4. update_exclusions の表示名→API名変換:
   - regionMapping[displayName] があればそれを使う（リージョン）
   - countryToIso[displayName] があればそれを使う（国名）
   - それ以外はそのまま（domestic系やPO Box）
5. console.logの出力はMCPのstdioと干渉するので、export_all_excelとimport_excelでは
   console.logを一時的にリダイレクトする（出力をキャプチャしてtext contentに含める）
