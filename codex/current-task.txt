## 問題
eBay APIの `GET /sell/account/v1/fulfillment_policy` のレスポンスで、
`shipToLocations.regionExcluded` の各エントリには `regionName` のみ存在し、
`regionType` は含まれていません。

実際のAPI応答例:
```json
{
  "regionExcluded": [
    {"regionName": "AF"},
    {"regionName": "Africa"},
    {"regionName": "Alaska/Hawaii"},
    {"regionName": "APO/FPO"},
    {"regionName": "AS"},
    {"regionName": "Central America and Caribbean"},
    {"regionName": "CN"},
    {"regionName": "Europe"},
    {"regionName": "FJ"}
  ]
}
```

しかし現在のコードは `r.regionType === 'COUNTRY_REGION'` 等で判定しているため、
全てスキップされて0件になっています。

## 修正方針
`regionName` から種類を推定するユーティリティ関数を作成し、各コマンドで使用する。

### 判定ロジック
1. `regionMapping` のキーまたは値に一致 → リージョン（COUNTRY_REGION）
   - "Africa", "Asia", "Central America and Caribbean", "Europe", "Middle East",
     "North America", "Oceania", "Southeast Asia", "South America"
   - およびeBay内部名: "Central_America_and_Caribbean", "Middle_East", "North_America", "Southeast_Asia", "South_America"
2. 特殊名 → その他
   - "Alaska/Hawaii", "APO/FPO", "US Protectorates" → domestic (STATE_OR_PROVINCE)
   - "PO Box" → other (PO_BOX)
3. 2文字のISO国コード（大文字） → 国（COUNTRY）
4. `countryToIso` の値に一致する2文字コード → 国（COUNTRY）
5. `countryToIso` のキーに一致する国名 → 国（COUNTRY）
6. それ以外 → 不明だが国として扱う

### 新規ファイル: src/lib/classify-region.ts

```typescript
import { regionMapping, countryToIso } from '../data/definitions';

export type RegionType = 'COUNTRY_REGION' | 'COUNTRY' | 'STATE_OR_PROVINCE' | 'PO_BOX';

// regionMappingのキーと値を両方含むSet
const regionNames = new Set([
  ...Object.keys(regionMapping),
  ...Object.values(regionMapping),
]);

// 特殊地域
const domesticRegions = new Set(['Alaska/Hawaii', 'APO/FPO', 'US Protectorates']);

// ISO国コードのSet（2文字大文字）
const isoCodes = new Set(Object.values(countryToIso));

// 国名のSet
const countryNames = new Set(Object.keys(countryToIso));

export function classifyRegionName(regionName: string): RegionType {
  if (regionNames.has(regionName)) return 'COUNTRY_REGION';
  if (domesticRegions.has(regionName)) return 'STATE_OR_PROVINCE';
  if (regionName === 'PO Box') return 'PO_BOX';
  if (/^[A-Z]{2}$/.test(regionName) && isoCodes.has(regionName)) return 'COUNTRY';
  if (/^[A-Z]{2}$/.test(regionName)) return 'COUNTRY'; // 2文字大文字はISOコードと推定
  if (countryNames.has(regionName)) return 'COUNTRY';
  return 'COUNTRY'; // デフォルトは国扱い
}
```

function regionExcludedToRules(regionExcluded: Array<{ regionName: string; regionType?: string }>): ExclusionRule[] {
  const rules: ExclusionRule[] = [];
  for (const r of regionExcluded || []) {
    const rType = r.regionType || classifyRegionName(r.regionName);
    if (rType === 'COUNTRY_REGION') {
      const regionDisplay = reverseRegionMapping[r.regionName] || r.regionName;
      rules.push({ type: 'region', value: regionDisplay, action: 'exclude', note: '' });
    } else if (rType === 'COUNTRY') {
      // ISOコードの場合は国名に変換して表示
      const isoToCountry: Record<string, string> = {};
      for (const [name, iso] of Object.entries(countryToIso)) {
        isoToCountry[iso] = name;
      }
      const displayName = isoToCountry[r.regionName] || r.regionName;
      rules.push({ type: 'country', value: displayName, action: 'exclude', note: '' });
    } else if (rType === 'STATE_OR_PROVINCE') {
      rules.push({ type: 'domestic', value: r.regionName, action: 'exclude', note: '' });
    } else if (rType === 'PO_BOX') {
      rules.push({ type: 'other', value: 'PO Box', action: 'exclude', note: '' });
    }
  }
  return rules;
}
