## 問題
現在の実装は312ポリシーに対して個別にgetFulfillmentPolicy()を呼んでいるため非常に遅い。
しかし、getFulfillmentPolicies()の一覧APIレスポンスに既にshipToLocations.regionExcludedが含まれている。

## 修正内容
getFulfillmentPolicy()の個別呼び出しループを削除し、
getFulfillmentPolicies()の結果だけで全データを取得する。

修正前（遅い）:
```typescript
for (let i = 0; i < policies.length; i++) {
  const p = policies[i];
  const detail = await getFulfillmentPolicy(opts.token, p.fulfillmentPolicyId);
  const excluded = detail.shipToLocations?.regionExcluded || [];
  // ...
}
```

修正後（高速）:
```typescript
for (let i = 0; i < policies.length; i++) {
  const p = policies[i];
  console.log(chalk.gray(`  [${i + 1}/${policies.length}] ${p.name}`));
  const excluded = p.shipToLocations?.regionExcluded || [];
  // ...以下同じ処理
}
```

つまり:
1. `getFulfillmentPolicy` のimportを削除（使わなくなるため）
2. ループ内の `await getFulfillmentPolicy(...)` を削除
3. `detail.shipToLocations` を `p.shipToLocations` に変更
4. それ以外のロジック（行の追加、スタイル等）はそのまま維持
