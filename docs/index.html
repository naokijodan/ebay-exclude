ebay-exclude

eBay発送除外国を一括管理するCLIツール

CSV管理 一括適用 差分更新 レート制限対応 Claude Code MCP連携

概要

ebay-exclude は、eBay の発送除外国・地域ポリシーを CSV で集中管理し、その内容をアカウントへ一括適用できる CLI ツールです。複数ポリシーの同期や差分の可視化、レート制限を考慮した安全な適用をサポートします。

  • CSV からの宣言的管理（Git で履歴管理が容易）
  • 既存設定のエクスポートと差分確認（セマンティック・ディフ）
  • ドライラン対応で安全に検証
  • レート制限・再試行・部分適用の考慮

セットアップ手順

3.1 前提条件

  • Node.js 18 以上
  • npm

3.2 インストール

cd ~/Desktop/ebay-exclude
npm install
npm run build

3.3 eBay Developer Account

  1. eBay Developer Program にアクセスし、アカウントを作成/ログイン
  2. Application を作成し、必要な API 権限を有効化
  3. OAuth Token の取得方法を確認（Authorization Code Flow を推奨）
  4. 本番利用の場合は Production 環境のクレデンシャルでトークンを取得

3.4 認証設定

トークンのスコープは発送ポリシーの読み書きに必要な範囲を付与してください。

方法1: 環境変数
export EBAY_TOKEN="your-oauth-token"
方法2: コマンドラインフラグ
ebay-exclude --token "your-oauth-token" list

コマンド一覧

4.1 init 4.2 list 4.3 export 4.4 diff 4.5 apply

4.1 init — テンプレートCSV生成

テンプレートとなる CSV をカレントディレクトリに作成します。

ebay-exclude init
ebay-exclude init -o my-exclusions.csv

4.2 list — ポリシー一覧表示

現在の発送ポリシーを一覧表示します。フィルタで特定のポリシーに絞り込み可能です。

ebay-exclude list
ebay-exclude list --filter "eco_new"

4.3 export — 現在の設定をエクスポート

既存の除外国設定を CSV に保存します。ポリシーIDを指定してエクスポートすることも可能です。

ebay-exclude export -o current.csv
ebay-exclude export --policy "POLICY_ID" -o current.csv

4.4 diff — 差分表示

CSV と現在の設定の差分を、セマンティックに表示します（追加/削除/更新の分類）。

ebay-exclude diff exclusions.csv
ebay-exclude diff exclusions.csv --filter "eco_new"
セマンティック・ディフ（例）
policy: eco_new
  include country JP  # unchanged
  - include country AU  # removed
  + include country NZ  # added
  ~ region Asia: exclude -> include  # updated

summary:
  added:    1
  removed:  1
  updated:  1
  unchanged: 1

4.5 apply — 一括適用

CSV の内容をアカウントへ適用します。ドライランで内容を確認してから本番適用するのが安全です。

# ドライラン（確認のみ）
ebay-exclude apply exclusions.csv --dry-run
# 実際に適用
ebay-exclude apply exclusions.csv
# フィルタ付き
ebay-exclude apply exclusions.csv --filter "eco_new"
# 強制更新（ステートファイル無視）
ebay-exclude apply exclusions.csv --force

CSV の編集方法

5.1 CSVフォーマット

列名説明値
typeルール種別region / country / domestic / other
value対象の値地域名 / ISOコード / domestic名 / other名
actionアクションexclude / include
noteメモ自由記述（任意）

5.2 基本的な使い方

type,value,action,note
region,Africa,exclude,アフリカ全域除外
region,Asia,exclude,アジア全域除外
country,AU,include,オーストラリアは発送可
country,JP,include,日本は発送可
domestic,Alaska/Hawaii,exclude,アラスカ・ハワイ除外

5.3 よくあるパターン

  • パターン1: 全地域除外 → 特定国のみ発送可
  • パターン2: 特定地域のみ除外（例: Middle East のみ exclude）
  • パターン3: 既存設定の微調整（差分のみ更新）

Claude Code 連携（MCP）

ebay-exclude は MCP（Model Context Protocol）サーバーとして Claude Code と連携できます。自然言語でポリシー管理が可能になります。

7.1 セットアップ

Claude Code の設定ファイル（~/.claude/settings.json）に以下を追加：

"mcpServers": {
  "ebay-exclude": {
    "command": "node",
    "args": ["/path/to/ebay-exclude/dist/mcp-server.js"]
  }
}

設定後、Claude Code を再起動してください。

7.2 利用可能なツール

ツール名説明話しかけ方の例
list_policiesポリシー一覧取得「ポリシー一覧を見せて」
get_policy_exclusions特定ポリシーの除外設定を取得「中古エコノミーの除外国は？」
update_exclusions1件のポリシーの除外設定を更新「このポリシーからアフリカを除外して」
bulk_update_exclusionsフィルタ付き一括除外更新「全ポリシーからMiddle Eastを除外して」
export_all_excel全ポリシーをExcel出力「全ポリシーをExcelに出力して」
import_excelExcelからインポート「このExcelの変更を反映して」

7.3 使い方の例

例1: ポリシー一覧の確認
ユーザー：
「eBayのポリシー一覧を見せて」
Claude Code：
→ list_policies を呼び出し、312件のポリシーを一覧表示
例2: 一括除外追加
ユーザー：
「全ポリシーからMiddle Eastを除外して」
Claude Code：
→ bulk_update_exclusions(addExclusions: ["Middle East"], dryRun: true)
→ プレビュー表示後、確認を経て dryRun: false で本番適用
例3: フィルタ付き更新
ユーザー：
「中古のポリシーだけ、アフリカを除外解除して」
Claude Code：
→ bulk_update_exclusions(filter: "中古", removeExclusions: ["Africa"], dryRun: true)
例4: Excel出力→編集→インポート
ユーザー：
「全ポリシーをExcelに出力して」
（Excelを手動で編集）
「ebay-policies.xlsx の変更を反映して」
Claude Code：
→ export_all_excel → ユーザーが編集 → import_excel(dryRun: true) → 確認後 import_excel(dryRun: false)

7.4 注意事項

  • bulk_update_exclusions と import_excel はデフォルトで dry-run（安全のため）。実際に反映するには確認後に dryRun: false で再実行
  • 国名・地域名は英語表記（例: Japan, Africa, Middle East, PO Box）
  • ポリシー名で指示された場合、Claude Code は自動的に list_policies でIDを特定してから操作します
  • 認証情報は .env ファイルから自動読み込み。毎回トークンを指定する必要はありません

FAQ

Q: トークンはどうやって取得する？
A: eBay Developer Program のアプリで OAuth の Authorization Code Flow を設定し、Production クレデンシャルでトークンを発行してください。取得後は環境変数または --token で指定します。
Q: 300件以上のポリシーがある場合は？
A: ページングに対応しています。list と export は自動で複数ページを取得します。パフォーマンスのため、--filter の併用を推奨します。
Q: レート制限に引っかかった場合は？
A: 自動リトライとバックオフを実装しています。apply 中に 429 が返った場合は一定時間待機し、未適用分から再開します。
Q: CSVに記載していない国はどうなる？
A: デフォルトでは非指定の項目は変更しません。完全同期したい場合は、diff の結果を確認し、必要に応じて除外や包含を CSV に明記してください。
Q: 既に include されている国を再度 exclude するには？
A: 同一対象に対して action を exclude に変更して CSV を用意し、apply を実行します。diff で更新として表示されます。

トラブルシューティング

エラー: "Missing token"
環境変数 EBAY_TOKEN を設定するか、コマンドの --token フラグで指定してください。
エラー: "Unknown region/country"
地域名/ISOコードを確認してください。地域は定義済み名、国は ISO 3166-1 alpha-2 を使用します。
エラー: "429 Too Many Requests"
一定時間待って再実行。ツールは自動バックオフしますが、--filter で対象を絞ると改善します。
エラー: "API 401 Unauthorized"
トークンの有効期限・スコープ・環境（Sandbox/Production）を確認し、必要に応じて再取得してください。
