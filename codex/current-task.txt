## 修正1: zodスキーマの追加

MCP SDKの `server.tool()` メソッドは第3引数にzodスキーマを受け取る:

```typescript
import { z } from 'zod';

server.tool(
  'tool_name',
  'description',
  { paramName: z.string().describe('説明') },  // ← zodスキーマ
  async (args) => { ... }
);
```

zodはインストール済み（@modelcontextprotocol/sdkの依存に含まれる）。

### 各ツールのスキーマ定義:

#### list_policies
```typescript
{ marketplaceId: z.string().optional().describe('eBay marketplace ID (default: EBAY_US)') }
```

#### get_policy_exclusions
```typescript
{ policyId: z.string().describe('フルフィルメントポリシーID') }
```

#### update_exclusions
```typescript
{
  policyId: z.string().describe('フルフィルメントポリシーID'),
  addExclusions: z.array(z.string()).optional().describe('追加する除外項目の表示名配列。例: ["Africa", "Japan", "PO Box"]'),
  removeExclusions: z.array(z.string()).optional().describe('解除する除外項目の表示名配列'),
  setExclusions: z.array(z.string()).optional().describe('除外リストを丸ごと置き換える表示名配列'),

```

```typescript

```

## 修正2: bulk_update_exclusions ツール追加

```typescript
server.tool(
  'bulk_update_exclusions',
  'ポリシー名フィルタで一括除外設定更新（部分一致）',
  {
    filter: z.string().optional().describe('ポリシー名の部分一致フィルタ（空なら全ポリシー対象）'),
    addExclusions: z.array(z.string()).optional().describe('追加する除外項目の表示名配列'),
    removeExclusions: z.array(z.string()).optional().describe('解除する除外項目の表示名配列'),
    dryRun: z.boolean().optional().default(true).describe('trueでプレビューのみ、falseで実際に反映'),
  },
  async (args) => {
    // 1. getFulfillmentPolicies() で全ポリシー取得
    // 2. filter があれば名前で部分一致フィルタ
    // 3. 各ポリシーに対して add/remove を適用
    // 4. 変更があるポリシーのみ更新（並列実行、最大5並列）
    // 5. 結果を返す: { totalMatched, updated, skipped, dryRun, details: [{name, id, added, removed}] }
  }
);
```

### 並列実行の実装:
p-limitパッケージがインストール済みなので使用:
```typescript
import pLimit from 'p-limit';
const limit = pLimit(5); // 最大5並列
const promises = policiesToUpdate.map(p => limit(() => updateSinglePolicy(p)));
await Promise.all(promises);
```

## 修正3: import_excel の並列化

import_excel ツール内で importAllCommand() を呼んでいるが、
src/commands/import-all.ts の forループを並列化する。

import-all.ts の修正:
- 既存の for ループ（逐次）を、p-limit(5) による並列処理に変更
- 結果の集計は Promise.all の後にまとめる

```typescript
import pLimit from 'p-limit';

// 変更前（逐次）:
// for (let rowNum = 2; rowNum <= sheet.rowCount; rowNum++) {
//   const row = sheet.getRow(rowNum);
//   ...
//   await updateFulfillmentPolicy(...);
// }

// 変更後（並列）:
const limit = pLimit(5);
const tasks: Array<() => Promise<void>> = [];

for (let rowNum = 2; rowNum <= sheet.rowCount; rowNum++) {
  const row = sheet.getRow(rowNum);
  const policyId = String(row.getCell(policyIdCol).value || '').trim();
  const policyName = String(row.getCell(1).value || '').trim();
  if (!policyId) continue;

  const regionExcluded: Array<{ regionName: string }> = [];
  for (const rc of regionColumns) {
    const cellValue = String(row.getCell(rc.colNum).value || '').trim().toLowerCase();
    if (cellValue === 'x' || cellValue === 'exclude') {
      const apiName = displayNameToApiName(rc.displayName);
      regionExcluded.push({ regionName: apiName });
    }
  }

  tasks.push(() => limit(async () => {
    const currentPolicy = await getFulfillmentPolicy(opts.token, policyId);
    const currentExcluded = currentPolicy.shipToLocations?.regionExcluded || [];
    const currentNames = new Set(currentExcluded.map((r: any) => r.regionName));
    const newNames = new Set(regionExcluded.map(r => r.regionName));

    const hasChanges = currentNames.size !== newNames.size ||
      [...currentNames].some(n => !newNames.has(n));

    if (!hasChanges) {
      skippedCount++;
      return;
    }

    // ... diff表示 ...

    if (!dryRun) {
      await updateFulfillmentPolicy(opts.token, policyId, {
        ...currentPolicy,
        shipToLocations: { ...currentPolicy.shipToLocations, regionExcluded },
      } as any);
    }
    updatedCount++;
  }));
}

await Promise.all(tasks.map(t => t()));
```
